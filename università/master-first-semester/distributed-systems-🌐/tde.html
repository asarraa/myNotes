  1. Consider a simple data store and two implementations: (a) the system is implemented using a single machine, (b) the
    system is replicated across 5 machines using the Raft consensus protocol to provide consistency across replicas.
    Compare the two implementations in terms of response time for client requests, consistency, fault-tolerance.

Response time for client requests: single machine is better than replicated state machine since the latter involve requests to the replicas, acks and then commits, while the first doesn't involve any communication since every occurs on the single machine. Consistency: In both cases consistency is assured, since on the single machine there's no replication there can't be any inconsistency, while in replicated state machine there's replication but the raft algorithm assures safety (consistency across replicas).

Fault-tolerance: with single machine there's none sinche if the machine fails the system is unreachable and stops working, while raft implements techniques for fault tolerance that in practise assure liveness.
