1 - Buffer Overflow

1.1


Prolog and Epilogue

BP (base pointer) or FP (frame pointer) --> pointer to a fixed location on a function's frame. From which actual parameters have positive offset, while local variables have negative offset.
SP --> address of the top of the stack
IP --> address of the current instruction being executed

ESP --> register where the SP is stored (either last cell or first free cell's address)
EBP --> register where the BP is stored
EIP --> register where IP is stored

We racall that as soon as a function call is executed:
1. Parameters are pushed on the stack, call assembly instruction is executed
2. Prelude is executed

Call
(pushes address of the next instruction to be executed after the called function returns. This will be the saved EIP or RET)
(moves address of the first instruction of the called function into EIP --> call function)

Prolog:
pushl %ebp ---> (push BP of caller into stack. This will be the saved EBP)
mov %esp, %esp ---> (save current SP into EBP register)
subl $SizeOfLocalVar, %esp --> (move SP down in order to make space for local variables)

Epilogue:
mov %ebp, %esp (current base is the new top of the stack)
pop %ebp (restore the saved EBP to the registry )
ret (pop saved EIP and jump there)

Where to put parameters and return address of called function after overwriting saved EIP

When we overwrite the EIP with the address of a function, we need to rember that the ret will pop that address and then the prelude of the function will be executed, pushing the EBP on the stack: that is, the cell of the overwritten EIP is where the EBP of the called function will be placed. So the simply: when we overwrite a saved EIP, that cell will contain the saved EBP of the called function, the cell above (toward higher addresses) will contain the saved EIP of the called function and the cell above again will contain the parameter to that function

2 - format strings vulnerability

Format strings vulnerability basics

In format string vulnerability we exploit printf(x) where we are able to define x.
We can start from how printf works:

  • According to the calling convention, the parameters for a printf call are pushed on the stack by the caller. The other arguments, such a pointer the format string itself, are pushed on the stack. Usually the format string itself is in some position in the stack, and what's pushed as argument a pointer to it.
    For example, if we have something like printf("AAAA %x %x", value1, value2). The stack will be

Thus, inserting %x in the printf without any corresponding parameter will result to printing whatever is on the stack, where it would expect its arguments to be.

Also, we can use special printf operator to print an arbitrary number of characters and to write something in the stack:

  • %n = write, in the address pointed to by the argument, the number of chars (bytes) printed so far. That is, it takes whatever is on the stack corresponding to its position (it works as %x, but the latter simply prints its argument out, %n does something else with it), interprets it as an address, goes to that location and write in that location the number of char printed so far. We need: address on the stack, corresponding to a target cell where to write into.
  • %[k]c: prints out a number k of characters.
    We can use this operator in order to easily decide the number of chars (bytes) printed so far, which is useful for the %n operator. In practice, the number of chars (bytes) printed so far is a very big number which corresponds to the shellcode we want to write on the target.
  • %[k]$x: allows to print the k-th parameter. For example %2$x write the same as the second %x in %x %x.
    We can use this operator to understand the pos between where the format string is located on the stack and the top of the stack (where we are.
  • Combining the first and the third, we get %[k]$n, which writes in the memory location whose address it finds in position k on the stack, the number of chars (bytes) printed so far

The attack: we want to write in the target memory cell, whose address is at distance pos from the top of the stack, a shellcode.
The exploit is a format string like:
address %[K]c %[pos]n

What happens if we pass it to a printf.

  • The format string itself will end up in some position in the stack.
  • the %[pos]n operator will at the address located at pos position in the stack (which we want to be our target), the number of characters length(address)+K, which we need to be our shellcode.
